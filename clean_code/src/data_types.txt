7) Систему локализации для мобильных приложений предпочтительней вынести на сервер.
Будет быстрая поддержка новых слов и просто исправление опечаток, без привязки к мобильному релизу в маркетах.
Тогда для мобильного приложения нужно будет отсылать локаль.
Для статичных имен, например, для пунктов меню, имена можно задавать локально.
Тогда для каждого языка создается файл строковых ресурсов. Каждая строка имеет id, по нему происходит получение текста.

8) Улучшил получение целого значения из полосы загрузки.
Вместо округления всегда в меньшую сторону целого числа, сделал округление к ближайшему целому.

9) Сравнение через "==" float типов. По меньшей мере лучше использовать статичный метод Float.compare.

10) Вынес в константы трешхолды, вроде минимальной-максимальной длины пароля: MIN_PASSWORD_LENGTH, MAX_PASSWORD_LENGTH.

11) В телеграмме есть обработчик нажатия на пункт меню в диалоговом окне вида if (which == 0) {...} else ...
Определенно стоит вынести номера позиций в константу, чтобы получился вид
switch(which) {
    case OPEN_ITEM: {...}
    case COPY_ITEM: {...}
    ...
}

12) В телеграмме получение key-value значений по ключу задается каждый раз строкой, вроде "archivehint".
Стоит часто используемые ключи вынести в константы.
