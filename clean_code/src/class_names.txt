/////////////3.2/////////////

fetch - get
// В одних случаях загрузка сущности осуществлялась вызовом метода get, в других = fetch.
// Для унификации получение сущности теперь будет осуществляться посредством вызова метода get.

send - new
// В одних случаях создание сущности идет через префикс send (sendMessage), в каких-то через create (createOffer).
// Для концепции созданий сущности пользователем будет использоваться префикс new (newMessage, newOffer, newPost).

setup - initialize
// Так же, в каких-то случаях инициализация объектов/зависимостей при открытии экрана происходит в методе setup, в каких-то - в init.
// Функция, в которой будут происходить такие операции, будет называться initialize.

find, search - seek
// Иногда функции поиска, например, в локальной базе называются find, для поиска на сервере - search.
// Более оптимальным будет название seek для поиска чего-либо (файл, пользователь, элемент массива).

onResult - onSuccess, onError
// По проекту комбинируется схема обработки ответа с сервера, когда в одних случаях вся логика заключена в onResult
// а в других - разделяется на отдельные методы onSuccess и onError.
// Оптимальным будет разделение на две функции, чтобы наглядно видеть, как обрабатывается конкретный результат.

open - navigateToScreen
// В проекте комбинировались названия функции на открытие экрана - open, navigate.
// Единый формат названия функций будет иметь префикс navigateTo и постфикс Screen. Например, navigateToProfileScreen.

open - show
// Также имели место всплывающие окна, которые отображались поверх текущего экрана.
// Это либо просто информативные окна, которые сами исчезают, либо диалоги, требующие ввода пользователя.
// Названия функций для такого сценария так же отличались (show, open).
// Для такого сценария функции должны иметь префикс show и постфикс Dialog/Popup. Например, showDiscountPopup.
